# 批判（#critique）

> 一句話定義：系統性地找出主張的強項與弱項，而不是全盤接受或全盤否定。

## 密涅瓦怎麼定義

Minerva 將 #critique 定義為「對作品、論證或方案進行建設性的系統評估」。屬於 **Formal Analyses（#formalanalyses）** 與 **Multimodal Communications（#multimodalcommunications）** 課程。重點：批判不是挑刺，而是平衡評估——優秀的批判先完整理解，再指出哪裡強、哪裡弱、可以如何改進。

## 核心概念

- **先理解再批判**：能用自己的話復述對方立場（甚至比對方說得更好）才有資格批判。
- **鋼人（steel man）vs. 稻草人**：批判最強版本的對方觀點，而不是最弱版本。
- **結構化批判**：評估前提、推理、證據、結論四層。
- **建設性取向**：指出問題後提出改進方向。
- **自我批判對稱**：對自己觀點使用同樣嚴格的批判標準。

## 典型應用場景

| 情境 | 怎麼用 |
|------|--------|
| 評審論文 | 優點、方法限制、替代解釋、後續研究建議——四段式。 |
| 讀書心得 | 這本書哪裡成立、哪裡跳躍、哪裡可推進？ |
| 給下屬回饋 | 做對什麼（具體） → 可改進什麼（具體） → 建議方向。 |
| 自我檢討 | 對自己的判斷用鋼人版對手論證交叉檢驗。 |

## 常見誤用

- **批判 = 反對**：把批判性思考誤解成「什麼都懷疑」。
- **稻草人批判**：攻擊對方最弱版本，容易但無價值。
- **情緒化批判**：個人好惡替代理性評估。
- **批判而不建設**：只說壞不提出改進方向。
- **不自我批判**：嚴以待人寬以律己。

## 如何自我練習

1. 批判任何東西之前，強迫自己寫出「對方最強版本」的論述。
2. 結構化練習：對每個主張列三個強項 + 三個弱項。
3. 對自己的筆記或文章做延遲批判——放一週後再讀。
