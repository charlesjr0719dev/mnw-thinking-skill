# 機率（#probability）

> 一句話定義：用 0 到 1 的數量表達不確定性，並根據機率法則正確進行推論。

## 密涅瓦怎麼定義

Minerva 將 #probability 定義為「應用機率原理來量化不確定性並進行理性推論」。屬於 **Formal Analyses（#formalanalyses）** 課程。Minerva 特別強調條件機率與貝氏更新，因為研究顯示這是人類直覺最不擅長的領域之一（Kahneman 的計程車問題、醫療檢驗問題即是經典）。

## 核心概念

- **加法法則 / 乘法法則**：獨立事件同時發生的機率相乘；互斥事件擇一的機率相加。
- **條件機率**：P(A|B) = P(A∩B) / P(B)。P(A|B) ≠ P(B|A)——這是人類直覺最大的陷阱。
- **貝氏定理**：新證據到來時如何更新先驗信念。
- **基率謬誤（base rate fallacy）**：忽略先驗機率，只看證據強度。
- **獨立性**：兩事件是否獨立，決定了能不能相乘。

## 典型應用場景

| 情境 | 怎麼用 |
|------|--------|
| 醫療檢驗 | 即使檢驗準確率 99%，若疾病盛行率 0.1%，陽性患病機率仍不到 10%。 |
| 投資組合 | 同產業股票相關高，不是獨立事件，不可直接相乘。 |
| 面試決策 | 即使「好候選人會這樣回答」是真的，你要的是「這樣回答的是好候選人」。 |
| 產品品管 | 兩道獨立檢驗都 95% 漏檢率 = 0.25% 整體漏檢率。 |

## 常見誤用

- **P(A|B) 當成 P(B|A)**：檢察官謬誤的根源。
- **忽略先驗**：只看證據強度，忘了事件本身有多罕見。
- **把相關事件當獨立**：多因素風險評估中最常見的錯。
- **把「不可能」與「很罕見」混用**：百萬分之一的事件放在地球尺度每天發生很多次。

## 如何自我練習

1. 練計程車問題、Monty Hall、醫療陽性三個經典，直到直覺與數學一致。
2. 看新聞時，對每個「機率」追問：分子是什麼？分母是什麼？條件是什麼？
3. 對自己的信念做「事前信心 %」記錄，事後校正。
