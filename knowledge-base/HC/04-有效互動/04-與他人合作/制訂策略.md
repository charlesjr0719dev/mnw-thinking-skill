# 制訂策略（#strategize）

> 一句話定義：為達成目標設計出整合資源、考量對手與環境的行動計畫。

## 密涅瓦怎麼定義
密涅瓦將此 HC 定義為：在資源有限、環境不確定、有競爭或合作對象的情境下，設計出可執行、可調整、具一致性的策略，而非單純的任務清單。屬於 Working Effectively with Others 子能力。

## 核心概念
- **策略 ≠ 計畫**：策略是「為何」與「如何取捨」，計畫是「做什麼」
- **Good Strategy**（Rumelt）三要素：診斷、指導方針、協調行動
- **取捨**是策略核心：不選擇什麼比選擇什麼更重要
- **OODA Loop**（Boyd）：Observe, Orient, Decide, Act——快速循環
- **策略層次**：目標 → 策略 → 戰術 → 行動
- 策略需考量對手反應（賽局思維）

## 典型應用場景
| 情境 | 怎麼用 |
|------|--------|
| 創業 | 選擇不服務某些客戶是策略的核心展現 |
| 職涯規劃 | 5 年目標 → 能力建構策略 → 年度行動 |
| 團隊協作 | 分工不只是任務分配，是策略性角色設計 |
| 學習新領域 | 先診斷自身起點，再設計學習路徑 |

## 常見誤用
- **壞策略**（Rumelt）：把「目標」當「策略」（「我們要成長 30%」不是策略）
- 列出所有想做的事，沒有取捨
- 策略與資源不匹配
- 一次定終身，不隨資訊更新
- 忽略對手或環境動態

## 如何自我練習
1. 對一個目標寫出：診斷（為何困難）、方針（核心選擇）、協調行動
2. 練習「策略倒推」：從結果逆推必要條件
3. 讀 Rumelt《Good Strategy Bad Strategy》、Porter《Competitive Strategy》
4. 每季檢視策略：資訊改變了嗎？方針還成立嗎？

## 與其他 HC 的關係
- [[領導原則]]：領導者的核心產出是策略
- [[複雜系統|系統動力]]：策略需考量系統反饋
- [[責任感]]：策略承諾意味責任承擔
- [[談判]]：談判策略是策略的子類
- [[一致性]]：策略的力量來自執行的一致
