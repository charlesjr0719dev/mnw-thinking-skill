# 構成（#composition）

> 一句話定義：在句子與段落層級建構清晰、流暢、無歧義的語言。

## 密涅瓦怎麼定義
#composition 聚焦在「微觀語言技藝」：如何組織句子結構、選擇時態、安排子句、避免冗贅，讓每一句話都承載意義。屬於 Multimodal Communications 中語言溝通的工藝型 HC，是書寫與口說的基本功。

## 核心概念
- **句子經濟學**：每個字都該有功能，刪除沒有也不影響的字
- **主動 vs. 被動**：主動更明確，被動用於凸顯受事者或隱藏施事者
- **平行結構**：並列元素採相同文法形式以減輕認知負擔
- **過渡與連貫**：用連接詞、代名詞、重複關鍵詞維持段內邏輯流
- **具象取代抽象**：以具體名詞與動詞取代抽象名詞化（nominalization）

## 典型應用場景
| 情境 | 怎麼用 |
|------|--------|
| 學術論文 | 刪除「很」「非常」等空洞副詞，讓論證緊緻 |
| 產品說明 | 用主動句「按此按鈕可儲存」而非「資料將被儲存」 |
| 簡歷條目 | 動詞開頭 + 量化結果，避免名詞堆砌 |
| 推特／短貼文 | 每字必斤斤計較，砍掉贅述 |

## 常見誤用
- 誤以為長句＝深刻，造成讀者解析困難
- 濫用被動語態以顯得客觀，實則模糊責任
- 平行結構不一致（「喜歡跑步、游泳、以及去騎腳踏車」）
- 過度名詞化：「進行討論」取代「討論」

## 如何自我練習
1. 用 Hemingway Editor 或朗讀法檢測句子可讀性
2. 逐句問：這句能否再砍 20% 字數？
3. 模仿範文——抄寫喜歡的作者段落，感受節奏與結構
4. 每週挑一段舊作改寫，對比前後的緊緻度

## 與其他 HC 的關係
- [[組織]]：構成處理句段層次，組織處理整體結構
- [[意涵]]：字詞選擇是構成的原料
- [[表達]]：口說的構成還要考量停頓與節奏
- [[正確性]]：構成的邏輯連貫支撐論證正確
