# 演算法（#algorithms）

> 一句話定義：把問題拆成有限、可執行、必定終止的步驟，讓解法可被重複且可被驗證。

## 密涅瓦怎麼定義
密涅瓦把 `#algorithms` 定義為「應用演算法思考策略來解問題並有效實作」：學生要能辨識問題類型、選對演算法範式（貪婪、分治、動態規劃、回溯）、分析複雜度。屬於 **Empirical Analyses** Cornerstone，在「問題解決」群。

## 核心概念
- **五要素**：輸入、輸出、明確性、有限性、可執行性。
- **範式**：貪婪、分治、動態規劃、回溯、分支定界。
- **複雜度分析**：時間 Big-O、空間 Big-O，知道規模成長時會不會爆。
- **正確性論證**：不變式（invariant）、歸納法、邊界條件。

## 典型應用場景
| 情境 | 怎麼用 |
|------|--------|
| 排練習時程 | 用動態規劃在 90 分鐘內最佳化練琴科目組合 |
| 查資料 | 二分搜尋：用有序索引把 1000 篇筆記查詢從 O(n) 降到 O(log n) |
| 安排旅行路線 | 旅行推銷員問題，近似演算法（2-opt）取代暴力解 |

## 常見誤用
- **先寫程式再想演算法**：結果複雜度災難級。
- **把演算法當萬靈丹**：模糊、需要人類價值判斷的問題硬套演算法。
- **忽略常數因子**：O(n) 不一定比 O(n log n) 快，資料小時常數決勝負。
- **不驗證正確性**：跑起來就以為對，漏掉邊界 case。

## 如何自我練習
1. 每學一個新演算法，手動在紙上跑 5 筆輸入，包含退化 case。
2. 解 LeetCode 題時先寫偽代碼與複雜度，再寫程式。
3. 把日常決策（購物清單排序、書桌整理）試著演算法化一次。
