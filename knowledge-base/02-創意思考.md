# 02-創意思考（Thinking Creatively）

## 在 Minerva 四大能力中的定位
Minerva 的四大核心能力是：**批判思考、創意思考、有效溝通、有效互動**。

創意思考在這個架構中扮演「**從無到有的產出引擎**」：當批判思考負責拆解、評估、挑毛病，創意思考則負責**發現新問題、提出新假說、生成新方案、驗證新發現**。兩者是思考的雙翼——沒有創意思考，批判只剩挑錯；沒有批判思考，創意只剩空想。

對應的 Cornerstone Course 是 **Empirical Analyses（實證分析）**，它教學生如何面對真實世界的資料與問題：先發掘什麼值得研究、再用研究方法驗證、最後把結果轉成可行的解。

## 三個子分類

### [[01-問題解決]]
**執行引擎**——當問題存在但解法未知時，你需要的心法與工具。
涵蓋：科學學習法、類比、限制條件、捷思法、演算法、最佳化、設計思考。

### [[02-發掘與探索]]
**發現引擎**——在還不知道問題長什麼樣時，把混沌變成可研究的對象。
涵蓋：建立假說、資料視覺化、建模。

### [[03-應用研究方法]]
**驗證引擎**——有了想法，怎麼設計研究去實際驗證並評估他人研究。
涵蓋：抽樣、觀察性研究、介入性研究、個案研究、訪談式研究、研究可複製性、比較組。

## 整體學習路徑建議
建議依「發現 → 解決 → 驗證」的自然研究循環來讀：

1. **第一圈：建立基礎**
   先讀 [[01-問題解決]] 中的 [[科學學習法]]，它是學其他 HC 的工具；再讀 [[02-發掘與探索]] 中的 [[資料視覺化]] 與 [[建立假說]]，建立從資料出發的思考習慣。

2. **第二圈：深入核心方法**
   精讀 [[限制條件]]、[[類比]]、[[建模]]；學會把問題形式化。

3. **第三圈：進入研究方法**
   完整讀 [[03-應用研究方法]]，把 [[抽樣]]、[[比較組]]、[[研究可複製性]] 當作日常讀論文與做決策的濾鏡。

4. **第四圈：整合應用**
   用 [[設計思考]] 把前述所有 HC 串成真實的產品或研究專案；做至少一個 end-to-end 的小研究，從假說到報告。

整個子分類最終的收穫：你能在任何情境下**自問自答**——
> 「我的問題是什麼？有沒有類似的舊題？限制是什麼？這個想法可以被證偽嗎？要怎麼設計比較組？結果能外推到哪裡？」

這組自問就是創意思考的底層反射。
